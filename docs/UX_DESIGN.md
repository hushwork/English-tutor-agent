## 2. 孩子端交互

### 2.1 LED 光环语言

LED 光环是孩子端**唯一的视觉反馈**。设计原则：
- 不占用中央视觉（孩子不需要"看"它）
- 颜色即信息（不需要识字）
- 动画传达状态变化

```
LED 光环规格:
  硬件: WS2812B × 24 环形 (或 ReSpeaker 自带 12 颗)
  位置: 设备正面，摄像头周围
  直径: ~80mm
```

#### 颜色-状态映射表

| 颜色 | 动画 | 亮度 | 状态 | 含义 | 持续时间 |
|------|------|------|------|------|---------|
| 🔵 蓝色 | 缓慢呼吸 (3s 周期) | 30% | 📹 记录仪 | 我在，但不打扰 | 持续（默认状态） |
| 🟢 绿色 | 恒亮 → 渐亮 1s | 60% | 🧑‍🏫 导师 (说话中) | Emma 正在说英语 | 说话期间 |
| 🟢 绿色闪烁 | 快闪 2 次 → 恒亮 | 80% | 🧑‍🏫 导师 (即将说话) | 我有话要说，注意 | 说话前 1 秒 |
| 🟡 黄色 | 跳跃式（弹跳感） | 70% | 🎮 游戏 | 来玩吧！ | 游戏期间 |
| 🟠 橙色 | 顺时针旋转流转 | 60% | 🤔 思考 | 正在理解你说的… | 推理期间 (1-2s) |
| 🟣 紫色 | 极暗微光 | 10% | 🌙 睡前 | 晚安，安静 | 睡前模式 |
| 🔴 红色 | 恒亮 | 40% | 🔇 休眠 | 镜头盖已盖 | 持续 |
| ⚪ 白色一闪 | 单次闪光 | 100% | 📸 拍照 | 看到新东西了！ | 0.3 秒 |

### 2.2 Emma 面部动画设计（Mid/Pro 版）

#### 设计原则

Emma 的面部是"可视化嘴巴"——不是角色扮演游戏，是发音教学工具。

| 规则 | 说明 |
|------|------|
| **只在说话时显示** | Emma 沉默时屏幕完全熄灭(AMOLED 纯黑) |
| **嘴型与语音精确同步** | 唇形跟随音素变化，类似视频通话中的嘴 |
| **非写实但清晰** | 卡通风格但嘴型夸张，便于孩子看清 |
| **目光"看着"孩子** | Emma 眼睛追踪摄像头方向，模拟眼神接触 |
| **表情辅助理解** | 疑问句→歪头+扬眉，鼓励→微笑，惊讶→张嘴 |

#### 技术方案

| 方案 | 技术 | 优势 | 劣势 |
|------|------|------|------|
| **Live2D** | 2D 骨骼动画 + 音素映射 | 轻量(< 50MB VRAM)、动作流畅、大量现成模型 | 仅 2D，不能旋转 |
| **VRM (3D)** | 3D 模型 + blendshape 动画 | 可旋转、可做深度感 | 需要 GPU 渲染 |
| **自定义 SVG** | 矢量图形 + 实时插值 | 零依赖、极轻量 | 需要自己设计和调试 |

推荐: **Live2D** — 对 Orin GPU 几乎无压力, 成熟音素映射工具链, 可以快速调整嘴型设计。

#### 音素-嘴型映射

```
音素     →  嘴型
/a/ /ae/ → 大开口(apple, cat)
/i/ /i:/ → 扁嘴微笑(bee, see)
/o/ /u/  → 圆唇(boat, blue)
/m/ /p/ /b/ → 闭唇(mom, pop, big)
/f/ /v/  → 下唇咬上齿(five, very)
/th/     → 舌尖碰牙(think, this)
/t/ /d/ /n/ /l/ → 舌尖顶上颚(top, dog)
默认     → 微张、放松
```

#### 显示状态切换

```
Emma 沉默     → 屏幕完全熄灭(AMOLED 黑色像素关闭)
Emma 即将说话 → 屏幕淡入(0.3s), 面部出现, 嘴微张
Emma 说话中   → 嘴型随音素实时变化
Emma 说完了   → 微笑 0.5s → 屏幕淡出(0.5s) → 全黑
Emma 听孩子说 → 微点头动画 + 偶尔眨眼(表示"我在听")
```

### 2.3 投影 AR 互动设计（Pro 版）

#### 投影元素设计规范

| 元素 | 大小(桌面) | 颜色 | 用途 |
|------|-----------|------|------|
| Emma 面部 | ~8cm 直径 | 自然肤色 | 视觉焦点，嘴型参考 |
| 单词卡片 | 5×3cm 圆角矩形 | 白底+黑字+彩色图标 | 教学展示/游戏选项 |
| 互动区高亮 | 10×10cm 虚线圈 | 半透明蓝色 | 提示孩子可放置物品的区域 |
| 粒子特效 | 散布 | 金色星星 | 正确反馈 |
| 进度条 | 15×1cm | 绿色渐变 | 游戏进度/星星收集 |

#### 手势交互定义

```
手势          系统理解                      触发动作
─────────────────────────────────────────────────────
单指指向      孩子在指某个投影元素           选中/高亮该元素
单指按住 1s   孩子想要"点击"确认             确认选择(触发)
手掌展开      孩子想展示物品/召唤 Emma       进入 Show&Tell 模式
挥手(左右)    孩子想翻页/切换                切换到下一组内容
双手比心      孩子很开心                    视觉特效反馈
两根手指      孩子在展示物品大小             触发尺寸相关教学
```

手势识别采用 MediaPipe Hands, 21 个手部关键点:

```
指尖(Index Tip) 坐标 → 映射到投影空间 → 判断与哪个 UI 元素重叠
重叠 + 停留 0.3s → 高亮
重叠 + 停留 1.0s → 触发
```

#### 投影-真实世界叠加规则

```
投影元素不覆盖真实物品:
  检测到真实绘本 → 投影内容偏移到绘本旁边
  检测到真实玩具 → 投影内容在玩具周围
  检测到孩子的手 → 投影内容在手下方(避免照到手背)

投影元素跟随真实物品:
  孩子移动绘本 → 投影高亮跟随移动
  孩子移动玩具车 → 投影道路跟随移动

投影增强而非替代:
  真实积木塔 → 投影数字标签("1, 2, 3 blocks!")
  真实画作 → 投影单词标注框
  真实绘本 → 投影当前句高亮


```python
# led_ring.py
import apa102  # ReSpeaker library

LED_COUNT = 12  # ReSpeaker has 12 APA102 LEDs

def set_breathing_blue(brightness=0.3):
    """Slow breathing effect — observing mode."""
    t = time.time()
    for i in range(LED_COUNT):
        val = int(128 + 127 * math.sin(t * 2.0 + i * 0.5))
        set_led(i, 0, 0, int(val * brightness))  # R=0, G=0, B=val

def set_solid_green():
    """Solid green — speaking mode."""
    fill_leds(0, 80, 0)

def flash_white():
    """Single white flash — photo moment."""
    fill_leds(255, 255, 255)
    time.sleep(0.1)
    fill_leds(0, 0, 0)
```

### 2.2 语音交互

#### Emma 的声音特征

| 参数 | 幼儿版 (3-6 岁) | 少儿版 (7-10 岁) |
|------|----------------|-----------------|
| **语速** | 正常语速的 70-80% | 正常语速的 85-95% |
| **音调** | 稍高，更夸张的音调变化 | 自然音调 |
| **平均句长** | 5-8 词 | 8-12 词 |
| **重复策略** | 新词重复 3 次（不同上下文） | 重复 1-2 次 |
| **鼓励频率** | 每 2-3 轮 | 每 4-5 轮 |
| **纠错方式** | 重述（不说"不对"） | 重述 + 偶尔温和提示 |

#### 声音设计细节

- **Qwen-Omni 原生语音**优先（自然、有情感）
- 睡前模式使用 edge-tts softer voice
- 单词强调：关键教学词拖长 1.5× 并稍提高音调
- 句子间停顿 0.8-1.2 秒（给孩子消化时间）

#### 唤醒-互动-退出的自然节奏

```
[孩子玩积木, Emma 沉默观察]
    ↓ 孩子举起红色积木看摄像头
[LED 绿色闪烁 2 次 — 准备说话]
    ↓ 0.5s
[LED 绿色恒亮 — 说话中]
Emma: "Wow! A red block! Can you find a blue one?"
    ↓ Emma 说完了
[LED 蓝色呼吸 — 返回观察]
    ↓ 孩子找到蓝色积木
[重复以上循环]
```

### 2.3 物理交互

| 交互 | 组件 | 操作 | 系统响应 |
|------|------|------|---------|
| **盖镜头盖** | 滑动式镜头盖 + 霍尔传感器 | 手动推上 | LED 变红色 → Emma: "Bye bye!" → 休眠 |
| **开镜头盖** | 同上 | 手动推开 | LED 变蓝色呼吸 → Emma: "I'm back!" |
| **调音量** | 旋转编码器 | 手动旋转 | 播放简短提示音（当前音量 10-100%） |
| **拍头顶/挥动手臂** | 摄像头动作检测 | 在摄像头前挥手 2 秒 | 进入 Show & Tell: "What do you want to show me?" |
| **说 "Emma!"** | 麦克风 + 唤醒词 | 清楚大声说 | LED 绿色 → "Yes? I'm here!" |

## 5. 错误与边界情况

### 5.1 孩子说"不要"或"Stop"

```
孩子: "Stop!" 或 "不要说话！" 或 "Go away!"

Emma: "Okay." → LED 变蓝色呼吸 → 进入 30 分钟静默期
30 分钟后自动恢复观察模式

如果 24 小时内孩子说了 3 次 "Stop"：
  → 降低介入频率到 1 次/小时
  → 家长收到通知："孩子今天似乎不太想互动，已自动降低介入频率"
```

### 5.2 Emma 听不懂孩子在说什么

```
孩子: (说了模糊不清的英语)
Emma: 听不清 → LED 橙色旋转 (思考)
Emma: "Hmm, can you say that again?"
      (2 次重试后)
Emma: "That's okay! Let's keep playing."
      → 回到观察模式
```

### 5.3 WiFi 断开

```
WiFi 断开 → LED 黄色闪烁 1 次 → 蓝色呼吸（纯本地模式）
家长收到通知（可选，网络恢复后推送）

孩子体验不变（本地 7B 模型继续工作）
云端功能降级通知（仅在家长 Dashboard 显示）
```

### 5.4 多个孩子同时在场

```
检测到 2+ 人：
  Emma: 不说话（不打扰社交互动）
  记录仪继续工作
  如果检测到英语对话 → 记录但不介入
```

## 6. 无障碍设计

| 考虑 | 设计 |
|------|------|
| **视觉障碍** | 纯语音交互（不依赖 LED），LED 只是锦上添花 |
| **听力障碍** | 暂时不适用（产品前提是听力正常） |
| **言语障碍** | 降低说话期望，更多旁白模式，不强迫孩子开口 |
| **ADHD 儿童** | 极低介入频率（1-2 次/小时），不打断注意力 |
| **自闭症谱系** | 完全可预测的 LED-语音对应，不意外打扰 |
