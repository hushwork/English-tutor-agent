# Camera Tutor — 硬件能力验证测试计划

> **目的**: 在正式开发前，验证硬件在目标场景下的性能是否满足设计要求
> **所需设备**: Jetson AGX Orin + C920 Webcam + ReSpeaker Mic Array + 音箱
> **测试环境**: 模拟幼儿房间 (15-20㎡，有玩具/绘本/书桌)

## 测试 1: 实时视频流性能

### 1.1 帧率和延迟

**目标**: 验证摄像头在 Orin 上的捕获帧率和编码延迟

```bash
# 测试脚本: tests/test_camera_perf.py
```

| 指标 | 目标 | 测量方法 |
|------|------|---------|
| 帧捕获帧率 | ≥ 10 fps (不做全量推理时) | OpenCV get(cv2.CAP_PROP_FPS) |
| 帧捕获帧率 | ≥ 3 fps (每帧送入推理时) | 实际 time.perf_counter() |
| 单帧 base64 编码 | < 50ms (1080p → JPEG → base64) | 计时 100 次取平均 |
| 内存占用 | < 500MB | nvidia-smi / jtop |

### 1.2 场景变化检测

**目标**: 验证场景变化检测算法能识别哪些是"有意义"的变化

**测试场景**:
- [ ] 孩子进入摄像头视野 → 检测到
- [ ] 孩子离开 → 检测到
- [ ] 孩子拿起新玩具 → 检测到
- [ ] 孩子翻开书 → 检测到
- [ ] 只有光线变化（窗帘被风吹动）→ 不应该触发
- [ ] 宠物/其他人经过 → 检测到但标记为"非主体"

**验收标准**: 有意义场景变化召回率 > 90%，误触发率 < 20%

**测试**: 音箱播放时，麦克风是否收到回声

## 测试 4: 端到端延迟

### 4.1 完整管线延迟

**这是整个硬件验证的核心测试**。

```
时间线测量:
  t0: 摄像头帧捕获
  t1: 场景分析（本地 YOLO / 变化检测）
  t2: base64 编码
  t3: Qwen-Omni 推理（本地 7B）
  t4: 语音合成/TTS 输出
  t5: 音箱播放完成
```

**分级目标**:

| 模式 | 目标延迟 (t5 - t0) | 可接受延迟 |
|------|-------------------|-----------|
| 物品识别 + 简单描述 | < 1.5s | < 2.5s |
| 复杂场景分析 + 对话 | < 3.0s | < 5.0s |
| 纯语音对话（无视觉） | < 1.0s | < 2.0s |

### 4.2 测试脚本

```python
# tests/test_e2e_latency.py

import time
import base64
import cv2
import httpx
from PIL import Image
import io

def measure_e2e():
    cap = cv2.VideoCapture(0)
    ret, frame = cap.read()
    t0 = time.perf_counter()

    # Encode
    _, buffer = cv2.imencode('.jpg', frame, [cv2.IMWRITE_JPEG_QUALITY, 80])
    b64 = base64.b64encode(buffer).decode()
    t1 = time.perf_counter()

    # Send to Omni server
    response = httpx.post(
        "http://localhost:8100/api/vision",
        json={"image_base64": b64, "prompt": "Describe what you see in one simple English sentence."},
        timeout=10
    )
    t2 = time.perf_counter()

    result = response.json()
    print(f"Encode: {(t1-t0)*1000:.0f}ms | Network+Infer: {(t2-t1)*1000:.0f}ms | Total: {(t2-t0)*1000:.0f}ms")
    print(f"Output: {result['text']}")
    cap.release()
```

## 测试 5: 散热与稳定性

### 5.1 7×24 压力测试

| 指标 | 测试方法 | 目标 |
|------|---------|------|
| 满载温度 | 连续推理 1 小时 | GPU < 85°C, CPU < 80°C |
| 空闲温度 | 待机 1 小时 (仅摄像头+麦克风采集) | GPU < 45°C |
| 温度波动 | 30 分钟满载 → 30 分钟空闲 → 循环 4 次 | 无降频、无卡顿 |
| 内存泄漏 | 24 小时连续运行 | 内存使用增长 < 10% |
| 风扇噪声 | 满载时在 1m 距离测量 | < 35dB (约等于图书馆背景) |

### 5.2 功耗测量

