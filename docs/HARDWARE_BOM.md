## 8. 显示模块（Mid / Pro 新增）

### 8.1 设计铁律：屏幕只是"可视化的嘴巴"

| 规则 | 说明 |
|------|------|
| **只在说话时亮** | Emma 沉默 → 屏幕物理背光关闭，完全黑暗 |
| **只显示面部** | 无 UI、无文字、无菜单、无可点击元素 |
| **暗色背景** | 背景深蓝/黑色，仅面部区域发光 |
| **嘴型优先** | 嘴唇区域对比度最高，吸引视觉注意 |
| **不可触摸** | 屏幕凹在设备内部/前方有物理分隔 |
| **不是玩具** | 不响应触摸，不可玩游戏，不可浏览内容 |

### 8.2 Mid 方案: 小型 AMOLED 屏（Emma 面部 + 嘴型）

**推荐: Waveshare 5.5" AMOLED (1920×1080)**

| 参数 | 规格 |
|------|------|
| **尺寸** | 5.5 寸（约手机大小，固定在设备正面） |
| **分辨率** | 1920×1080 |
| **面板** | AMOLED（纯黑不发光 — 暗色场景几乎零光污染） |
| **接口** | HDMI (通过 Orin HDMI 输出) |
| **亮度** | 350 nit（室内足够，不刺眼） |
| **视角** | 178°（孩子从下方看清 Emma 面部） |
| **价格** | ¥250-350 |

**为什么是 AMOLED 而非 LCD？**
- LCD 暗色背景仍有背光泄漏（孩子能感知到"屏幕亮着"）
- AMOLED 纯黑 = 像素完全关闭 = 物理不发光
- 只有 Emma 面部（约占屏幕 40% 面积）在发光
- 剩余 60% 纯黑区域零光输出 → 低幼友好

**备选: 7" IPS LCD (1024×600)**

| 参数 | 规格 |
|------|------|
| **价格** | ¥150-250 |
| **劣势** | IPS 暗色有背光泄漏 |
| **选择条件** | 成本优先，用软件补偿（最低背光 + 暗色 UI） |

### 8.3 Pro 方案: Pico 投影仪（桌面 AR 互动空间）

在 Pro 方案中，投影仪**取代 LCD 屏** — 不仅显示 Emma 面部，还在桌面投射互动元素。

**正式方案: Philips PicoPix Max / AAXA P8 / XGIMI Magic Cube**

| 参数 | 规格 |
|------|------|
| **分辨率** | 1080p (1920×1080) |
| **亮度** | 300-500 ANSI 流明（室内桌面投影可用） |
| **投射比** | 1.2:1（50cm 距离 → 约 40×25cm 画面，覆盖桌面活动区） |
| **对焦** | 自动对焦（安装后免调） |
| **梯形校正** | 自动垂直梯形校正（从上方投影到桌面的畸变修正） |
| **接口** | HDMI / USB-C |
| **功耗** | 15-30W（Orin USB-C 直供或独立供电） |
| **噪声** | < 30dB（需实测是否被麦克风拾取） |
| **价格** | ¥2,500-4,000 |

**验证方案（低成本）: 便携微投**

| 参数 | 规格 |
|------|------|
| **型号** | 淘宝"便携投影仪"（200-300 ANSI 流明） |
| **分辨率** | 854×480 / 1280×720 |
| **价格** | ¥500-800 |
| **劣势** | 分辨率低、亮度低（较暗环境适用）、无自动梯形校正 |
| **场景** | 初期验证手势追踪+投影联动可行性，成本优先 |

### 8.5 手势追踪（Pro 版核心技术）

**方案 A: 纯摄像头 + MediaPipe Hands（推荐，零额外硬件）**

| 参数 | 详情 |
|------|------|
| **框架** | MediaPipe Hands (Google) |
| **精度** | 21 个手部关键点，手指尖定位误差 < 1cm |
| **延迟** | < 30ms（Orin 本地推理） |
| **硬件** | 已有 C920 摄像头（无需额外投入） |
| **局限** | 仅覆盖摄像头视野范围 |

```
摄像头帧 → MediaPipe Hands
              ↓
         21 个手部关键点 (x,y,z)
              ↓
         指尖坐标 → 映射到投影坐标系
              ↓
         指尖在哪个投影元素上方？
         距离 < 阈值 = 触发"触摸"事件
```

**方案 B: 深度相机（备选，精度更高）**

| 参数 | 详情 |
|------|------|
| **型号** | Intel RealSense D435 / Orbbec Astra |
| **精度** | 毫米级深度，可区分"悬停 vs 触摸" |
| **价格** | ¥1,000-2,000 |
| **选择条件** | 如果 MediaPipe 精度不够（孩子手指细、误触多） |
